◆ AlCan_Dev_DisplayFrame()

AL_S32 AlCan_Dev_DisplayFrame ( AL_CAN_FrameStruct *  Frame)

This function validates the frame pointer to ensure it is not NULL. It then logs the frame's ID, data length code (DLC), actual data length in bytes, and whether the ID is extended, the frame is remote, and the bit rate is switched. It iterates through the data words of the frame, logging each one. The function concludes by logging a completion message.

Parameters
FramePointer to the CAN frame structure to be displayed.
Returns
AL_OK on successful execution, AL_CAN_ERR_NULL_PTR if the frame pointer is NULL.
Examples:

AL_S32 AlCan_Dev_RecvFrame ( AL_CAN_DevStruct *  Dev,
AL_CAN_FrameStruct *  Frame 
)

This function checks the device and frame pointers for nullity, the device's state for readiness, and the receive buffer for non-emptiness. It then reads the control word and the data from the receive buffer, populates the frame structure accordingly, and releases the receive buffer. If the frame contains an error or if the receive buffer is found empty after attempting to release it, appropriate error codes are returned.

Parameters
DevPointer to the CAN device structure.
FramePointer to the structure where the received frame will be stored.
Returns
AL_OK on success, error code otherwise.
